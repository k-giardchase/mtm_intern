JIRA Basics Review

Run through this checklist to see if you are familiarized enough with JIRA to accomplish the list below:

  1. Login to our JIRA website
  2. Find five bugs on the metaltoad-dev.metaltoad.com site. Log all five by using JIRA capture to create an issue with the type "Bug" and the priority set to minor. Assign all issues to the QA Onboarding Project.
  3. Create an issue with identical properties (Assigned project, priority, issue type etcâ€¦) to step 2 using JIRA capture that contains a screenshot that has been annotated
  4. Using the search feature locate the two issues you created (Use the more option to filter by creator)
  5. Save that specific search criteria as a filter that you can later refer back to when creating different issues (Information on filters and saving searches is located in the search for issues section of the JIRA 101 guide)

If you are able to do all these things you are now ready to manually test and report defects on the different projects your team is working on!
